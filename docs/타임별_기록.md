#### 3. 버그 수정

##### React key 중복 에러
- **문제:** `<br>` 태그의 key가 중복되어 콘솔 에러 발생
- **해결:** `charIdRef.current++`로 각 문자 추가 시마다 고유 ID 생성

##### 애니메이션 반복 실행
- **문제:** 2단, 3단 진입 시 **매번** 애니메이션이 실행되어 깜빡임 발생
- **해결:** 
  - `hasLeftAboutRef`, `hasLeftAppRef` 플래그 추가
  - **첫 진입 시:** 플래그가 `false`이므로 애니메이션 실행 안 함
  - **재진입 시:** 플래그가 `true`이므로 애니메이션 실행 → 플래그 `false`로 리셋

##### 새로고침 시 푸터로 스크롤
- **문제:** 브라우저가 이전 스크롤 위치 복원, URL 해시(#footer 등)로 인해 하단으로 이동
- **해결:**
  - `useEffect`에서 `window.location.hash` 확인 후 `window.history.replaceState`로 해시 제거
  - `window.scrollTo(0, 0)`로 맨 위로 강제 이동

### 주요 코드 구조

#### app/page.tsx 핵심 로직
```typescript
// 타자기 효과
const { displayText, showCursor, restart } = useTypewriter(text, delay);

// IntersectionObserver (1단 예시)
useIntersectionObserver(
  heroRef,
  0.2,
  () => {
    if (hasLeftHeroRef.current) {
      // 재진입 시에만 애니메이션
      heroRef.current?.classList.remove("hero-play");
      requestAnimationFrame(() => {
        requestAnimationFrame(() => {
          heroRef.current?.classList.add("hero-play");
          restart();
        });
      });
      hasLeftHeroRef.current = false;
    }
  },
  () => {
    // 이탈 시 플래그 설정
    hasLeftHeroRef.current = true;
    heroRef.current?.classList.remove("hero-play");
  }
);

// 페이지 로드 시 맨 위로 + 해시 제거
useEffect(() => {
  if (window.location.hash) {
    window.history.replaceState(null, "", window.location.pathname);
  }
  window.scrollTo(0, 0);
}, []);
```
